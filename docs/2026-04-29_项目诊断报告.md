## 三、🔴 严重 Bug（必须立即修复）

### Bug 1：`calculate_kdj` 重复定义

**位置**：`src/indicators.py` 第 94 行 vs 第 170 行

```python
# ── 第 94 行：类静态方法（正确）─────────────────────────────
@staticmethod
def calculate_kdj(df: pd.DataFrame, n=9, m1=3, m2=3) -> pd.DataFrame:
    low_n  = df["最低"].rolling(window=n, min_periods=1).min()
    ...

# ── 第 170 行：重复定义（会覆盖前者！）──────────────────────
def calculate_kdj(df: pd.DataFrame, n=9, m1=3, m2=3) -> pd.DataFrame:
    low_n  = df["最低"].rolling(window=n, min_periods=1).min()
    ...  # 逻辑完全相同
```

**影响**：Python 按顺序执行，第 170 行的函数会覆盖第 94 行的静态方法。任何调用 `TechnicalIndicators.calculate_kdj()` 的代码都会报 `TypeError`。

**修复**：删除第 170 行开始的重复函数定义。

### Bug 2：`backtest_web.py` 早停机制完全失效

**位置**：`backtest_web.py` 中 XGBoost 模型训练部分

```python
# ❌ 当前代码（错误）
model.fit(
    X_tr, y_tr,
    eval_set=[(X_va, np.zeros(len(X_va)))],  # 验证集标签全传 0！
    verbose=False
)

# ✅ 修复后
y_va = df.iloc[va_idx]["target"].values
model.fit(X_tr, y_tr, eval_set=[(X_va, y_va)], verbose=False)
```

**影响**：`early_stopping_rounds=30` 的目的是"验证集 loss 不降就停"，但你传了一组 `[0,0,0,...]` 作为验证集标签。XGBoost 计算的验证 loss 永远是错的，早停永远不触发。模型会在训练集上过拟合 30 轮以上。

**严重性**：高 — 模型实际精度会显著低于评估指标显示的值。

### Bug 3：`backtester.py` 回测数据污染

**位置**：`src/backtester.py` 第 ~80 行

```python
# ❌ 当前代码（硬编码，导致基本面和资金面在回测中失效）
tmp_data["indicators"] = {
    "gross_margin": 20, "net_margin": 10, "roe": 8,    # 全部=常数
    "revenue_growth": 10, "profit_growth": 10,
}
tmp_data["pe"] = None
tmp_data["pb"] = None
tmp_data["main_net_flow"] = 0      # 资金面=0！
tmp_data["chip_signals"] = []       # 筹码面=空！
```

**影响**：回测实际上只验证了**技术面 + 情绪面**两个因子。基本面和资金面的 IC/ICIR 数据全部是假的——它们在回测中永远输出 50 分基准分。

**修复方向**（选一）：
- **方案A（推荐）**：从 `hist_daily.db` 历史数据库读取财务数据用于回测
- **方案B**：明确标注"本次回测仅覆盖技术面"，仅用 `tech_score` 和 `sentiment_score` 评估模型

## 四、🟡 中等优先级问题

### 问题 4：`compute_factor_ic` 数据不足

`parameter_optimizer.py` 的 IC 计算依赖 `reports/` 目录下的 `*_report.json` 文件：
- 需要 ≥30 个日报文件才能得出统计显著的 IC 值
- 当前 reports 目录文件数量不明（需要用户确认）
- IC < 0.05 的因子建议淘汰，IC 0.05~0.10 保留降权，IC > 0.10 保留升权

**验证命令**：
```bash
ls ~/Desktop/quant_trading/reports/*_report.json | wc -l
```

---

### 问题 5：XGBoost 与规则模型未融合

`optimize_xgb_model.py` 独立训练、保存到 `config/xgb_models/*.json`，但 `backtest_web.py` 没有调用这些模型。两条路线完全平行：
- **规则路线**：ai_scorer.py → 输出 0~100 概率
- **XGBoost 路线**：optimize_xgb_model.py → 独立训练，从不参与评分

**建议**：在 `backtest_web.py` 中，对已有专属 XGBoost 模型的股票优先使用 XGB 预测，没有的才用规则评分。
